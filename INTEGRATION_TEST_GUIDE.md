# 🚀 [팀원2-장민재] 전체 통합 프론트/백엔드 로컬 테스트 가이드

안녕하세요 팀원 여러분! 팀원2(장민재)입니다.
현재 깃허브 `dev` 브랜치의 최신 상태(팀원들의 프론트엔드 모놀리식 구조)를 모두 제 브랜치로 가져온 다음, 그 거대한 `app.py` 안에 **제가 담당한 냉장고 DB 로직(Blueprint)만 안전하게 끼워 넣는 통합 작업**을 완료했습니다!

이제 기존처럼 **단 1개의 서버(`python app.py`)만 켜면** 팀원들이 만든 모든 UI 화면들과, 제가 만든 진짜 DB 연동 냉장고 기능이 완벽하게 하나의 사이트처럼 작동합니다.

제 브랜치(`local_jang`)에서 모든 테스트가 완료되었으니, 아래 순서대로 여러분의 컴퓨터에서 직접 실행해 보시고 정상 작동하는지 확인해 주세요!

---

## 🛠️ 통합 테스트 환경 세팅 방법 (처음 1회만)

**1. 제 최신 코드 가져오기**
```powershell
git fetch origin
git merge origin/local_jang
```

**2. 가상환경 및 라이브러리 세팅**
```powershell
# 가상환경 켜기
.\venv\Scripts\activate

# 의존성 라이브러리 최신화 (SQLAlchemy 추가됨)
pip install -r requirements.txt
```

**3. 원격 DB(cloud33)에 내 냉장고 테이블 생성하기**
*(이전과 동일하게 최상단에 .env 파일이 있어야 합니다!)*
```powershell
python -c "from app import create_app, db; app=create_app(); app.app_context().push(); db.create_all()"
```

---

## 💻 서버 실행 및 화면 확인하기

**1. 서버 실행**
팀원들이 기존에 쓰던 방식 그대로 `app.py`를 실행합니다.
```powershell
python app.py
```

**2. 브라우저 접속 테스트**
👉 **http://127.0.0.1:5000/** 

* 홈 화면, 로그인, 레시피 추천 화면 등 다른 팀원분들의 UI가 정상적으로 렌더링됩니다.
* 네비게이션에서 **"내 냉장고"** 탭을 누르거나 `http://127.0.0.1:5000/fridge` 로 이동하시면,
  * 기존의 더미 데이터가 아닌, **실제 cloud33 MariaDB**와 연동된 진짜 재료 데이터가 표시됩니다!
  * 재료를 추가하거나 삭제하면 DB에 즉시 반영되며, D-Day 계산도 정상 작동합니다.
